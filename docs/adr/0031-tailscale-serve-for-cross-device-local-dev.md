# Tailscale serve for cross-device local dev access

To test `web`'s local dev server from other devices on the same tailnet (a phone, another laptop), `web:start` additionally runs `tailscale serve --bg --https=8080 https+insecure://127.0.0.1:8080` (task `root:setup-tailnet`, wired as a `deps:` of `web:start`), reusing the already-persistent `desktop-oscar` Tailscale node rather than standing up a separate ephemeral node or sidecar container. It's reachable at `https://desktop-oscar.wolf-triggerfish.ts.net:8080`, restricted to specific devices via the tailnet's ACLs (configured in the Tailscale admin console, not in this repo). `https+insecure://` is required because the internal proxy hop from `tailscaled` to the mkcert-backed HTTPS backend doesn't share Tailscale's trusted cert chain; this only affects that local loopback hop, not the cert actually served across the tailnet, which is a real Let's Encrypt-backed certificate issued by Tailscale.

This is purely additive to the existing mkcert + hosts-file loopback setup (see [ADR-0029](0029-local-dev-hostname-convention.md), [ADR-0030](0030-web-dev-server-https-only.md)), which remains the primary, offline-capable path: day-to-day local dev at `localhost.www.craftsmans-ledger.dev:8080` does not depend on Tailscale being up, authenticated, or reachable.

Teardown is a separate, manually-run task (`root:teardown-tailnet`, using the exact-flags-plus-`off` form: `tailscale serve --https=8080 https+insecure://127.0.0.1:8080 off`), not `tailscale serve reset`. `desktop-oscar` is a shared, persistent identity also used for other homelab services on the same tailnet, and `reset` clears all serve configuration on the node, not just this one mapping.
